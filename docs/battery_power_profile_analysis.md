# 储能电站充放电功率曲线分析报告

**数据来源**：`source_data/日清算结果查询电厂侧(1)_副本.xlsx`  
**分析区间**：2026-01-27 ~ 2026-04-17（共 81 天，排除 1/25-1/26 调试日）  
**目的**：从实际清算数据中提取充放电功率随时段变化的规律，用于改进充放电策略模型。

---

## 1. 核心结论

> **充电需要 5 小时窗口，而非 4 小时，才能完成约 800 MWh 的充电量。**  
> 充放电功率在首尾时段存在明显折扣（爬坡/收尾效应），中间时段接近额定功率。

---

## 2. 充电功率曲线分析

### 2.1 事件时长分布

识别到主充电事件（`充电_实际用电量 > 10 MWh/h` 的连续小时段）共 **145 次**：

| 时长 | 次数 | 占比 |
|------|------|------|
| 2小时 | 48 | 33% |
| 3小时 | 26 | 18% |
| **5小时** | **56** | **39%** |
| 4小时 | 6  | 4%  |
| 6小时 | 10 | 7%  |

> **主流为 5 小时充电事件（39%），是最能代表正常运营的充电模式。**  
> 2小时事件多为补充充电或调试，不代表完整充电周期。

### 2.2 标准 5 小时充电功率曲线（基于 56 次事件均值）

| 时段位置 | 均值（MWh） | 中位数（MWh） | 占额定200MW比例 | 范围 |
|----------|------------|--------------|----------------|------|
| 第1小时（爬坡）| **129.5** | 145.8 | 64.8% | 14~161 |
| 第2小时（满功率）| **193.1** | 195.5 | 96.6% | 148~200 |
| 第3小时（满功率）| **194.3** | 195.3 | 97.1% | 170~200 |
| 第4小时（满功率）| **193.1** | 195.1 | 96.6% | 162~200 |
| 第5小时（收尾）| **94.8**  | 90.6  | 47.4% | 12~164  |
| **合计** | **804.8** | 822.3 | **100.6%** | — |

> 5 小时充电合计 **804.8 MWh ≈ 800 MWh**，与额定容量吻合。

### 2.3 归一化功率比例（以事件内最大小时量为1.0）

所有长度 ≥ 2 的充电事件中，按相对位置统计：

| 位置 | 均值比例 | 中位数比例 | 25% | 75% | 样本数 |
|------|---------|----------|-----|-----|--------|
| first（首小时） | 0.641 | 0.732 | 0.457 | 0.795 | 137 |
| inner（中间各小时）| 0.983 | 0.999 | 0.998 | 1.000 | 250 |
| last（末小时） | 0.529 | 0.461 | 0.295 | 0.783 | 137 |

### 2.4 建议的梯形充电功率函数

```
P_charge(slot, n_slots_total) =
  P_max × α_up       if slot == 0          （第1小时，爬坡）
  P_max × α_full     if 0 < slot < n-1     （中间小时，满功率）
  P_max × α_down     if slot == n-1        （末小时，收尾）
```

**实测参数（基于5小时事件均值）：**
- `P_max`   = 195 MWh/h（实测满功率，而非额定200 MW）
- `α_up`   = 0.664（第1小时：~130 MWh）
- `α_full` = 0.990（中间小时：~193 MWh）
- `α_down` = 0.486（末小时：~95 MWh）

**策略含义：** 选取连续 **5 小时充电窗口**（而非 4 小时），可实现约 800 MWh 充电。

### 3.3 放电功率归一化（首/中/末位置比例）

| 位置 | 均值比例 | 中位数比例 | 样本数 |
|------|---------|----------|--------|
| first（首小时） | 0.660 | 0.652 | 145 |

### 3.4 建议的梯形放电功率函数

```
P_discharge(slot, n_slots_total) =
  P_max × β_up       if slot == 0          （第1小时，爬坡）
  P_max × β_full     if 0 < slot < n-1     （中间小时）
  P_max × β_down     if slot == n-1        （末小时）
```

**参考参数：**
- `β_up`   = 0.660（首小时约 132 MWh）
- `β_full` = 0.990（中间小时约 198 MWh）
- `β_down` = 0.860（末小时约 172 MWh）

## 4. 对策略模型的影响

### 当前模型（固定4小时窗口）
```
充电成本 = avg_charge_price × 800 MWh
放电收入 = avg_discharge_price × 800 MWh
```
选取预测价格最低连续 4 小时充电，最高连续 4 小时放电。

### 改进方案（功率加权窗口）

**充电侧：** 改为 **5 小时窗口**，使用功率权重计算加权平均电价：
```python
CHARGE_WEIGHTS = [0.664, 0.990, 0.996, 0.990, 0.486]  # 5个小时的功率权重
# 充电加权电价 = sum(price_h × w_h) / sum(w_h)
# 充电总量固定 = 800 MWh（sum(w_h × P_max) ≈ 800 MWh）
```

**放电侧：** 仍可用 4 小时窗口（放电首末折扣不对称，但末尾折扣较小），  
或同样扩展为 5 小时窗口以提高一致性：
```python
DISCHARGE_WEIGHTS = [0.660, 0.990, 0.990, 0.990, 0.860]  # 可选
```

**收益计算调整：**
```
充电成本 = Σ(actual_price_h × energy_h)  # 逐小时实际结算
放电收入 = Σ(actual_price_h × energy_h)
```
其中 `energy_h` 按功率曲线权重分配，总和约等于 800 MWh。

---

## 5. 数据局限性说明

1. **充电侧**: 5小时事件中第1小时均值 129.5 MWh，离散度较大（14~161 MWh），可能受日前调度计划影响，实际爬坡不完全一致。
2. **放电侧**: 放电高度分散（多数为1-2小时短事件），难以用单一连续窗口描述；策略模型中仍建议保持连续窗口假设，接受一定误差。
3. **额定功率**: 满功率实测约为 **195 MWh/h**（而非额定200 MW），可能受联络线容量限制。

## 6. 建议实现方式

```python
# 充电功率曲线（5小时，单位: 相对权重）
CHARGE_POWER_PROFILE = [0.664, 0.990, 0.996, 0.990, 0.486]
CHARGE_WINDOW_H = 5   # 充电窗口改为5小时

# 放电功率曲线（4小时或5小时）
DISCHARGE_POWER_PROFILE_4H = [0.660, 0.990, 0.990, 0.860]
DISCHARGE_WINDOW_H = 4  # 放电窗口保持4小时

def weighted_avg_price(prices_h: np.ndarray, weights: list) -> float:
    """以功率权重计算加权平均电价（用于窗口比较）"""
    w = np.array(weights)
    p = np.array(prices_h)
    return float(np.dot(w, p) / w.sum())

def actual_energy_cost(prices_h: np.ndarray, weights: list, total_mwh: float) -> float:
    """计算实际充放电费用"""
    w = np.array(weights) / np.sum(weights)
    return float(np.dot(w, prices_h) * total_mwh)
```
