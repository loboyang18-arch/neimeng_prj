## 1 电站基本参数


| 参数              | 数值                         |
| --------------- | -------------------------- |
| 装机功率            | 200 MW（额定），最大充/放 195 MW    |
| 额定容量            | 800 MWh                    |
| 双程效率（η_RT）      | 91.0%                      |
| 单程效率（η_1way）    | √0.91 ≈ 95.39%             |
| 充电爬坡速率（ΔP_RAMP） | 65 MW / 15min（约 3 个时段爬满功率） |
| 日维护及生活用电        | 13.03 MWh / 天（按日均节点价结算）    |
| 日最大充电量          | 1200 MWh（1.5 × 容量）         |
| 最短连续充/放时长       | 1 小时（4 个 15 分钟时段）          |


**效率建模说明**：

- 充电侧：电网侧输入 c[t] MW，电池存入 ηc·c[t] MW（单程效率损耗）。
- 放电侧：电网侧获得 d[t] MW，电池需释放 d[t]/ηd MW（单程效率损耗）。
- 双程示例：充入 800 MWh → 存 762.8 MWh → 放出 727.6 MWh，整体 91.0%。

## 3 充放电决策模型（MILP）

### 3.1 变量定义

每天独立求解，时间粒度 **15 分钟**，每天 T = 96 个时段（Δt = 0.25 h）。


| 变量     | 类型  | 含义           | 范围       |
| ------ | --- | ------------ | -------- |
| c[t]   | 连续  | 充电功率（MW）     | [0, 195] |
| d[t]   | 连续  | 放电功率（MW）     | [0, 195] |
| yc[t]  | 二值  | 本时段是否在充电     | {0, 1}   |
| yd[t]  | 二值  | 本时段是否在放电     | {0, 1}   |
| soc[t] | 连续  | 时段末荷电状态（MWh） | [0, 800] |


变量总数：5 × 96 = **480 个**（含 192 个二值变量）。

### 3.2 目标函数

```
最小化：Σ p̂[t] · c[t] · Δt  −  Σ p̂[t] · d[t] · Δt
         t=0..95                  t=0..95
```

其中 `p̂[t]` 为预测节点电价（元/MWh），使用小时级预测值按每小时 4 时段平铺展开。

**跨日 SOC 模式**下，额外增加终端价值项，引导合理存留：

```
目标函数 += − p̂_次日均 · ηd · α_disc · soc[T−1]
```

其中 α_disc = 0.95（折扣系数，防止无限囤积）。

### 3.3 约束体系

#### ① SOC 动态方程

```
soc[t] = soc[t−1] + ηc · c[t] · Δt − d[t] · Δt / ηd    （t = 0, …, 95）
```

初始条件：`soc[−1] = soc_init`（日清零模式下为 0；跨日模式下为前一天日末 SOC）。

#### ② SOC 边界

```
0 ≤ soc[t] ≤ 800 MWh        （所有时段）
soc[T−1] = 0                 （仅日清零模式，或仿真最后一天）
```

#### ③ 不同时充放

```
yc[t] + yd[t] ≤ 1            （所有时段，禁止同时充放）
```

#### ④ 功率-二值联动

```
c[t] ≤ 195 · yc[t]           （充电功率上限与二值联动）
d[t] ≤ 195 · yd[t]           （放电功率上限与二值联动）
```

#### ⑤ 功率爬坡约束

```
|c[t] − c[t−1]| ≤ 65 MW     （充电爬坡速率限制）
|d[t] − d[t−1]| ≤ 65 MW     （放电爬坡速率限制）
```

#### ⑥ 充放切换最短间隔（≥ 15 分钟）

```
yc[t] + yd[t+1] ≤ 1         （充电结束后至少间隔 1 个时段再放电）
yd[t] + yc[t+1] ≤ 1         （放电结束后至少间隔 1 个时段再充电）
```

#### ⑦ 最小连续运行时长（≥ 1 小时 = 4 时段）

若在时段 t 启动充电（yc[t]=1 且 yc[t−1]=0），则后续 3 个时段必须保持充电：

```
−yc[t+k] + yc[t] − yc[t−1] ≤ 0    （k = 1, 2, 3；线性化，无需辅助变量）
```

对 yd 同样适用。边界处理：t=0 时 yc[−1]=0 隐含。

#### ⑧ 日充电量上限

```
Σ c[t] · Δt ≤ 1200 MWh      （每日充电总量不超过 1.5 × 容量）
t=0..95
```

### 3.4 跨日 SOC 模式

**启用方式**：运行时加 `--carry_soc` 参数。

**动机**：当预测电价偏低（或实际出现负价）时，不强制当天放完，将剩余电量留至次日高价时段消纳，规避"负价禁止放电"导致计划失败的问题。

**终端价值设计**：

- 每 MWh 剩余电量的估值 = 次日预测均价 × 单程效率（ηd）× 折扣系数（0.95）
- 若今日价格高于此估值 → 优化器倾向今日放完；反之则留存
- 仿真最后一天强制 soc[T−1] = 0，避免虚拟收益

**PF 基准在跨日模式下**同样启用跨日 SOC，以已知次日实际均价作为终端价值，确保比较公平。

## 4 回测方法

### 4.1 数据源


| 数据     | 来源文件                                                                 | 粒度    |
| ------ | -------------------------------------------------------------------- | ----- |
| 预测节点电价 | `output/experiments/v8.0-jan25-sudun500/test_predictions_hourly.csv` | 小时    |
| 实际节点电价 | `source_data/日清算结果查询电厂侧(1)_副本.xlsx` → `一期_省内实时节点电价`                  | 15 分钟 |
| 实际运营收益 | `output/actual_spot_revenue_jan27_apr17.csv`                         | 日     |


### 4.2 回测流程

```
对每个交易日 d：
  1. 读取小时级预测价格（24 个值），平铺为 96 个 15 分钟值
  2. 读取当日真实 15 分钟节点电价（96 个值）
  3. 用预测价格求解 MILP → 充放电计划 (c[t], d[t])
  4. 用实际价格评估收益：
       毛收益 = Σ p_actual[t] × d[t]×Δt − Σ p_actual[t] × c[t]×Δt
       辅助用电成本 = avg(p_actual) × 13.03 MWh
       净收益 = 毛收益 − 辅助用电成本
  5. （同步）用实际价格再次求解 MILP → 完全预知基准 (c_pf, d_pf)
     计算 PF 净收益作为理论上限
  6. 兑现率 = 净收益 / PF净收益
  7. 跨日模式：soc[T-1] 传入第 d+1 天作为 soc_init
```

**求解器**：`scipy.optimize.milp`（HiGHS 后端），每天时限 60 秒。

### 4.3 基准对照


| 基准           | 说明                                           |
| ------------ | -------------------------------------------- |
| **完全预知 PF**  | 用真实电价求解同一 MILP，代表理论最优上限                      |
| **启发式方案B**   | 选连续 4 小时均价最低/最高窗口，含 89.9% 效率与 13.03 MWh 辅助成本 |
| **小时级 MILP** | 同一 MILP 框架但 24 时段粒度（无爬坡/最小运行约束）              |
| **实际现货收益**   | 从日清算文件计算的电站真实结算收益（2026-01-27 ～ 04-17）        |

### 5.3 与实际运营对比

```
统计区间：2026-01-27 ～ 2026-04-17（81 天重叠期）

                   策略（跨日SOC）    实际现货    差额
净收益合计             1164.0 万      807.3 万   +356.7 万

全年外推                0.534 亿      0.364 亿  +0.170 亿
```

**实际运营落后于策略的主要原因**（推测）：

1. 实际调度依赖人工经验或简单规则，未充分利用价差信息
2. 实际调度受电网调度指令约束，不能完全自主选择时段
3. 部分日期出现充放电不全（如 1 月 25-26 日调试期）
4. 实际存在额外的爬坡等待和申报时间损耗
